# Сайт компании «Стройсервис» (Уфа)

Современный, быстрый и адаптивный одностраничный лендинг для компании **«Стройсервис»** (бывш. «Артметалл»), специализирующейся на изготовлении и монтаже навесов и металлоконструкций в Уфе и Республике Башкортостан.

Сайт разработан на стеке: **Next.js (App Router) + TypeScript + Tailwind CSS (v4)**.

---

## 🚀 Как запустить проект

Для работы с проектом вам понадобится установленный **Node.js** (рекомендуемая версия v20+ / v24+).

### 1. Установка зависимостей
Установите необходимые npm-пакеты:
```bash
npm install
```

### 2. Запуск в режиме разработки
Запустите локальный сервер разработки:
```bash
npm run dev
```
Сайт будет доступен по адресу [http://localhost:3000](http://localhost:3000).

### 3. Сборка для продакшена
Для компиляции оптимизированного статически сгенерированного билда выполните:
```bash
npm run build
```

## 🛠️ Руководство по редактированию и администрированию

Вся логика и контент структурированы так, чтобы вносить изменения было максимально легко. Основные данные вынесены в файлы конфигурации.

### 📞 Где менять контакты (телефон, соцсети, адрес)
Все глобальные контакты компании (номер телефона, ссылки на WhatsApp/Telegram, рабочий адрес, график работы) находятся в файле:
📌 `src/data/data.ts` в объекте `CONTACTS_DATA`:
```typescript
export const CONTACTS_DATA = {
  companyName: "Стройсервис",
  oldCompanyName: "Артметалл",
  phoneDisplay: "+7 (987) 246-12-34", // Измените отображаемый телефон
  phoneValue: "tel:+79872461234",     // Измените телефон для вызовов
  whatsappUrl: "https://wa.me/79872461234", // Ссылка на WhatsApp
  telegramUrl: "https://t.me/stroyservice_ufa", // Ссылка на Telegram
  address: "г. Уфа, ул. Производственная, д. 12/2",
  workHours: "Пн-Вс: 09:00 - 20:00 (без выходных)",
  email: "info@stroiservice-ufa.ru",
};
```

### ✍️ Где менять тексты (услуги, преимущества, FAQ, география)
Вся текстовая информация вынесена в массивы внутри того же файла:
📌 `src/data/data.ts`
* `SERVICES_DATA` — 8 основных услуг, буллеты-преимущества и теги.
* `ADVANTAGES_DATA` — преимущества («Почему выбирают нас»).
* `WORK_STEPS_DATA` — этапы работы («Как мы работаем»).
* `GEOGRAPHY_DATA` — перечень обслуживаемых городов (Уфа, Стерлитамак, Салават, Нефтекамск, Бирск) и примечания.
* `FAQ_DATA` — 8 часто задаваемых вопросов и ответов.

### 🖼️ Где менять фотографии портфолио
Фотографии для портфолио настраиваются в массиве `PORTFOLIO_DATA` в файле:
📌 `src/data/data.ts`

В карточках по умолчанию используются уникальные интерактивные векторные чертежи-плейсхолдеры в инженерном стиле (с меткой «Фото скоро»), что позволяет сайту выглядеть готовым и дорогим до загрузки реальных фото.
Чтобы заменить плейсхолдер на реальную фотографию:
1. Поместите файлы изображений (например, `project1.jpg`) в папку `public/`.
2. Измените поле `imagePlaceholder` нужного проекта в `PORTFOLIO_DATA` на путь к картинке:
```typescript
{
  id: "port-1",
  title: "Арочный навес для 2 автомобилей",
  type: "Навес из поликарбоната",
  city: "г. Уфа (п. Нагаево)",
  description: "Спроектирован и установлен просторный навес над парковочной зоной...",
  details: ["Размер: 6 х 6 м", "Материал: Сотовый поликарбонат 10 мм", "Срок реализации: 6 дней"],
  imagePlaceholder: "/project1.jpg" // Укажите путь к вашей фотографии в папке public
}
```

### ✉️ Где подключать отправку заявок (Telegram, CRM, Email)
Обработчик отправки формы находится в компоненте формы заказа:
📌 `src/components/OrderForm.tsx` в методе `handleSubmit`.

В коде подготовлены специальные подписанные блоки для быстрой интеграции:
* **Telegram-бот**:
```typescript
const tgBotToken = 'YOUR_TELEGRAM_BOT_TOKEN';
const tgChatId = 'YOUR_TELEGRAM_CHAT_ID';
const message = `🔔 Новый лид!\n👤 Имя: ${formData.name}\n📞 Тел: ${formData.phone}\n🏗️ Конструкция: ${formData.constructionType}\n💬 Коммент: ${formData.comment}`;
await fetch(`https://api.telegram.org/bot${tgBotToken}/sendMessage`, {
   method: 'POST',
   headers: { 'Content-Type': 'application/json' },
   body: JSON.stringify({ chat_id: tgChatId, text: message })
});
```
* **CRM (AmoCRM, Битрикс24)**: Отправка JSON-данных методом POST на адрес вашего вебхука.
* **Email-рассылка**: POST-запрос на ваш API-роут для отправки писем через SMTP/Nodemailer.

## 📈 SEO и Оптимизация

* **Семантическая разметка**: На странице используется строго один тег `<h1>`, логичная иерархия тегов `<h2>`-`<h4>`.
* **Быстрота**: Все компоненты оптимизированы для статического рендеринга (SSG), что дает 100 баллов в Google PageSpeed Insights.
* **Микроразметка**: Интегрирована микроразметка **LocalBusiness** и **Organization** в формате JSON-LD (в файле `src/app/layout.tsx`) для отображения красивого сниппета в поисковиках Яндекс и Google.
* **Файлы для поисковых систем**: Автоматически генерируются `robots.txt` и `sitemap.xml`.
